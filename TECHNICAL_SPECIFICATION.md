# Техническое задание проекта Tunuk - Beauty Services Platform

## 1. Общая информация о проекте

### 1.1. Название проекта
**Tunuk** - Платформа для домашних услуг в Центральной Азии

### 1.2. Описание проекта
Веб-платформа для предоставления выездных бьюти-услуг и других бытовых сервисов на дому. Платформа объединяет клиентов и профессиональных мастеров, обеспечивая удобный процесс поиска услуг, бронирования и взаимодействия.

### 1.3. Целевая аудитория
- **Клиенты**: Жители Центральной Азии, нуждающиеся в бьюти-услугах на дому
- **Мастера**: Профессиональные специалисты в области красоты и бытовых услуг
- **Администраторы**: Управляющие платформой

### 1.4. Технологический стек

#### Backend
- **Framework**: FastAPI (Python)
- **ORM**: SQLAlchemy 2.0+
- **База данных**: SQLite (разработка) / PostgreSQL (продакшн)
- **Аутентификация**: JWT (JSON Web Tokens)
- **Хеширование паролей**: bcrypt
- **Валидация**: Pydantic
- **Миграции**: Alembic

#### Frontend
- **Framework**: React 18
- **Сборщик**: Vite
- **Маршрутизация**: React Router DOM
- **HTTP клиент**: Axios
- **Иконки**: Lucide React, React Icons
- **Работа с датами**: date-fns

## 2. Функциональные требования

### 2.1. Система аутентификации и авторизации

#### 2.1.1. Регистрация пользователей
- **Функция**: Регистрация новых пользователей в системе
- **Поля формы**:
  - Email (обязательно, уникальный)
  - Телефон (обязательно, уникальный)
  - Полное имя (обязательно)
  - Пароль (обязательно)
  - Роль (по умолчанию: CLIENT, возможные: CLIENT, PROFESSIONAL, ADMIN)
- **Валидация**:
  - Проверка уникальности email и телефона
  - Валидация формата email
  - Хеширование пароля перед сохранением
- **API Endpoint**: `POST /api/auth/register`

#### 2.1.2. Вход в систему
- **Функция**: Аутентификация пользователей
- **Метод**: OAuth2 Password Flow
- **Поля**: Email (username), Пароль
- **Результат**: JWT токен доступа
- **Срок действия токена**: 30 минут
- **API Endpoint**: `POST /api/auth/login`

#### 2.1.3. Получение текущего пользователя
- **Функция**: Получение информации о текущем авторизованном пользователе
- **Требования**: Действующий JWT токен
- **API Endpoint**: `GET /api/auth/me`

#### 2.1.4. Роли пользователей
- **CLIENT** (Клиент): Базовый пользователь, может бронировать услуги
- **PROFESSIONAL** (Мастер): Может создавать и управлять услугами, принимать бронирования
- **ADMIN** (Администратор): Полный доступ к управлению системой

### 2.2. Управление пользователями

#### 2.2.1. Просмотр списка пользователей
- **Функция**: Получение списка пользователей с фильтрацией
- **Фильтры**:
  - По роли (CLIENT, PROFESSIONAL, ADMIN)
  - По минимальному рейтингу (для мастеров)
  - Пагинация (skip, limit)
- **API Endpoint**: `GET /api/users`

#### 2.2.2. Просмотр списка мастеров
- **Функция**: Получение списка мастеров с высоким рейтингом
- **Фильтры**:
  - Минимальный рейтинг (по умолчанию 4.8)
  - Только активные пользователи
  - Пагинация
- **API Endpoint**: `GET /api/users/professionals`

#### 2.2.3. Просмотр профиля пользователя
- **Функция**: Получение детальной информации о пользователе
- **Информация**:
  - Основные данные (имя, email, телефон)
  - Роль и статус активности
  - Для мастеров: рейтинг, количество отзывов, опыт, биография, фото профиля
- **API Endpoint**: `GET /api/users/{user_id}`

#### 2.2.4. Обновление профиля
- **Функция**: Редактирование собственного профиля
- **Доступные поля**:
  - Полное имя
  - Телефон
  - Биография (для мастеров)
  - Годы опыта (для мастеров)
  - Фото профиля (URL)
- **API Endpoint**: `PUT /api/users/me`

### 2.3. Управление услугами

#### 2.3.1. Категории услуг
- **beauty** - Бьюти-услуги (макияж, прически)
- **spa** - Спа-процедуры
- **massage** - Массаж
- **haircut** - Стрижка
- **nail_care** - Уход за ногтями
- **cleaning** - Уборка
- **repair** - Ремонт
- **other** - Прочее

#### 2.3.2. Просмотр каталога услуг
- **Функция**: Получение списка доступных услуг
- **Фильтры**:
  - По категории
  - По мастеру (professional_id)
  - Только активные услуги
  - Пагинация
- **API Endpoint**: `GET /api/services`

#### 2.3.3. Получение категорий
- **Функция**: Получение списка всех доступных категорий услуг
- **API Endpoint**: `GET /api/services/categories`

#### 2.3.4. Просмотр деталей услуги
- **Функция**: Получение полной информации об услуге
- **Информация**:
  - Название (многоязычное: name, name_ru, name_ky)
  - Описание (многоязычное: description, description_ru, description_ky)
  - Категория
  - Цена
  - Длительность (в минутах)
  - Информация о мастере
  - Изображение услуги
- **API Endpoint**: `GET /api/services/{service_id}`

#### 2.3.5. Создание услуги (только для мастеров)
- **Функция**: Добавление новой услуги мастером
- **Обязательные поля**:
  - Название (name)
  - Категория (category)
  - Цена (price)
  - Длительность (duration_minutes)
- **Опциональные поля**:
  - Многоязычные названия (name_ru, name_ky)
  - Описания (description, description_ru, description_ky)
  - URL изображения (image_url)
- **Автоматически**: professional_id устанавливается из текущего пользователя
- **API Endpoint**: `POST /api/services`

#### 2.3.6. Обновление услуги
- **Функция**: Редактирование существующей услуги
- **Права доступа**: Только владелец услуги (мастер) или администратор
- **API Endpoint**: `PUT /api/services/{service_id}`

#### 2.3.7. Деактивация услуги
- **Функция**: Скрытие услуги из каталога (мягкое удаление)
- **Права доступа**: Только владелец услуги или администратор
- **API Endpoint**: `DELETE /api/services/{service_id}`

### 2.4. Система бронирования

#### 2.4.1. Статусы бронирования
- **pending** - Ожидает подтверждения
- **confirmed** - Подтверждено
- **in_progress** - В процессе выполнения
- **completed** - Завершено
- **cancelled** - Отменено

#### 2.4.2. Создание бронирования
- **Функция**: Бронирование услуги клиентом
- **Обязательные поля**:
  - ID услуги (service_id)
  - Дата и время бронирования (booking_date)
  - Адрес (address)
  - Телефон для связи (phone)
- **Опциональные поля**:
  - Детали адреса (address_details)
  - Примечания (notes)
- **Автоматически**:
  - client_id - из текущего пользователя
  - professional_id - из услуги
  - total_price - из цены услуги
  - status - "pending"
- **Валидация**:
  - Услуга должна быть активна
  - Мастер должен быть активен
- **API Endpoint**: `POST /api/bookings`

#### 2.4.3. Просмотр бронирований
- **Функция**: Получение списка бронирований
- **Фильтрация по роли**:
  - Клиенты видят только свои бронирования
  - Мастера видят только бронирования своих услуг
  - Администраторы видят все бронирования
- **Фильтры**:
  - По статусу
  - Пагинация
- **API Endpoint**: `GET /api/bookings`

#### 2.4.4. Просмотр деталей бронирования
- **Функция**: Получение полной информации о бронировании
- **Информация**:
  - Данные клиента и мастера
  - Информация об услуге
  - Дата и время
  - Адрес и контакты
  - Статус
  - Цена
  - Примечания
- **Права доступа**: Только участники бронирования (клиент или мастер)
- **API Endpoint**: `GET /api/bookings/{booking_id}`

#### 2.4.5. Обновление бронирования
- **Функция**: Изменение данных бронирования
- **Права доступа**: Клиент или мастер (участники бронирования)
- **Изменяемые поля**:
  - Дата и время
  - Адрес и детали адреса
  - Телефон
  - Статус (только для мастера/администратора)
  - Примечания
- **API Endpoint**: `PUT /api/bookings/{booking_id}`

#### 2.4.6. Отмена бронирования
- **Функция**: Отмена бронирования
- **Ограничения**:
  - Нельзя отменить завершенное бронирование
  - Нельзя отменить уже отмененное бронирование
- **API Endpoint**: `POST /api/bookings/{booking_id}/cancel`

---

### 2.5. Система отзывов и рейтингов

#### 2.5.1. Создание отзыва
- **Функция**: Оставление отзыва о мастере после завершения услуги
- **Требования**:
  - Только для завершенных бронирований (status = "completed")
  - Только клиент, который делал бронирование
  - Один отзыв на одно бронирование
- **Поля**:
  - ID бронирования (booking_id)
  - Рейтинг (rating) - от 1 до 5
  - Комментарий (comment) - опционально
- **Автоматически**:
  - client_id - из текущего пользователя
  - professional_id - из бронирования
  - Пересчет рейтинга мастера
- **API Endpoint**: `POST /api/reviews`

#### 2.5.2. Просмотр отзывов
- **Функция**: Получение списка отзывов
- **Фильтры**:
  - По мастеру (professional_id)
  - Пагинация
- **API Endpoint**: `GET /api/reviews`

#### 2.5.3. Просмотр деталей отзыва
- **Функция**: Получение информации об отзыве
- **API Endpoint**: `GET /api/reviews/{review_id}`

#### 2.5.4. Система рейтингов мастеров
- **Автоматический расчет**:
  - Средний рейтинг на основе всех отзывов
  - Количество отзывов
  - Обновление при добавлении/удалении отзыва
- **Требования к мастерам**:
  - Минимальный рейтинг 4.8 для отображения в списке профессионалов

### 2.6. Кабинет мастера

#### 2.6.1. Дашборд мастера
- **Статистика услуг**:
  - Общее количество активных услуг
- **Статистика бронирований**:
  - Всего бронирований
  - Завершенных
  - Ожидающих подтверждения
  - За последние 30 дней
- **Финансовая статистика**:
  - Общий доход (сумма завершенных бронирований)
- **Рейтинг**:
  - Средний рейтинг
  - Количество отзывов
- **API Endpoint**: `GET /api/professional/stats`

#### 2.6.2. Управление услугами мастера
- **Просмотр услуг**: `GET /api/professional/services`
- **Создание услуги**: `POST /api/professional/services`
- **Обновление услуги**: `PUT /api/professional/services/{service_id}`
- **Деактивация услуги**: `DELETE /api/professional/services/{service_id}`

#### 2.6.3. Управление бронированиями мастера
- **Просмотр бронирований**: `GET /api/professional/bookings`
- **Изменение статуса**: `PUT /api/professional/bookings/{booking_id}/status?status={status}`
  - Доступные переходы:
    - pending → confirmed (подтверждение)
    - confirmed → in_progress (начало работы)
    - in_progress → completed (завершение)

#### 2.6.4. Просмотр отзывов о мастере
- **Функция**: Просмотр всех отзывов, оставленных клиентами
- **API Endpoint**: `GET /api/professional/reviews`

#### 2.8.5. Управление отзывами
- **Просмотр всех отзывов**: `GET /api/admin/reviews`
  - Фильтры: по мастеру
- **Удаление отзыва**: `DELETE /api/admin/reviews/{review_id}`
  - Автоматический пересчет рейтинга мастера при удалении

### 2.9. Пользовательский интерфейс (Frontend)

#### 2.9.1. Публичные страницы
- **Главная страница** (`/`):
  - Приветственный экран
  - Популярные услуги
  - Топ мастера
  - Информация о платформе

- **Каталог услуг** (`/services`):
  - Список всех услуг с фильтрацией по категориям
  - Поиск услуг
  - Карточки услуг с изображениями

- **Детали услуги** (`/services/:id`):
  - Полная информация об услуге
  - Информация о мастере
  - Кнопка бронирования (требует авторизации)

- **Список мастеров** (`/professionals`):
  - Список мастеров с рейтингом ≥4.8
  - Фильтрация и сортировка
  - Карточки мастеров

- **Профиль мастера** (`/professionals/:id`):
  - Информация о мастере
  - Список услуг мастера
  - Отзывы клиентов
  - Рейтинг и статистика

#### 2.9.2. Страницы аутентификации
- **Вход** (`/login`):
  - Форма входа (email, пароль)
  - Ссылка на регистрацию

- **Регистрация** (`/register`):
  - Форма регистрации
  - Выбор роли (клиент/мастер)
  - Валидация полей

#### 2.9.3. Защищенные страницы
- **Профиль** (`/profile`):
  - Редактирование личных данных
  - Просмотр статистики

- **Мои заказы** (`/bookings`):
  - Список всех бронирований пользователя
  - Фильтрация по статусам
  - Детали каждого заказа

- **Форма бронирования** (`/bookings/new/:serviceId`):
  - Выбор даты и времени
  - Ввод адреса
  - Контактная информация
  - Примечания

#### 2.9.4. Кабинеты пользователей
- **Кабинет клиента** (`/client`):
  - Дашборд со статистикой
  - Мои заказы
  - Мои отзывы
  - Избранные мастера

- **Кабинет мастера** (`/professional`):
  - Дашборд со статистикой
  - Управление услугами (CRUD)
  - Управление бронированиями
  - Просмотр отзывов

- **Админ-панель** (`/admin`):
  - Дашборд со статистикой системы
  - Управление пользователями
  - Управление услугами
  - Управление бронированиями
  - Управление отзывами

#### 2.9.5. Компоненты интерфейса
- **Навигация (Navbar)**:
  - Логотип и название
  - Публичные ссылки (Услуги, Мастера)
  - Условные ссылки для авторизованных пользователей
  - Кнопки входа/выхода

- **Футер (Footer)**:
  - Контактная информация
  - Ссылки на важные страницы
  - Информация о платформе

- **Защищенные маршруты (ProtectedRoute)**:
  - Проверка авторизации
  - Проверка роли (для админ-панели)
  - Редирект на страницу входа при отсутствии доступа

- **Переключатель мок-данных (MockDataToggle)**:
  - Для разработки и демонстрации

## 4. Структура базы данных

### 4.1. Таблица users (Пользователи)
- `id` - Primary Key
- `email` - Email (уникальный, индексированный)
- `phone` - Телефон (уникальный, индексированный)
- `full_name` - Полное имя
- `hashed_password` - Хешированный пароль
- `role` - Роль (CLIENT, PROFESSIONAL, ADMIN)
- `is_active` - Статус активности
- `rating` - Рейтинг (для мастеров)
- `total_reviews` - Количество отзывов (для мастеров)
- `bio` - Биография (для мастеров)
- `experience_years` - Годы опыта (для мастеров)
- `profile_image` - URL фото профиля
- `created_at` - Дата создания
- `updated_at` - Дата обновления

### 4.2. Таблица services (Услуги)
- `id` - Primary Key
- `name` - Название (английский)
- `name_ru` - Название (русский)
- `name_ky` - Название (кыргызский)
- `description` - Описание (английский)
- `description_ru` - Описание (русский)
- `description_ky` - Описание (кыргызский)
- `category` - Категория
- `price` - Цена
- `duration_minutes` - Длительность в минутах
- `professional_id` - Foreign Key → users.id
- `image_url` - URL изображения
- `is_active` - Статус активности
- `created_at` - Дата создания

### 4.3. Таблица bookings (Бронирования)
- `id` - Primary Key
- `client_id` - Foreign Key → users.id
- `professional_id` - Foreign Key → users.id
- `service_id` - Foreign Key → services.id
- `booking_date` - Дата и время бронирования
- `address` - Адрес
- `address_details` - Детали адреса
- `phone` - Телефон для связи
- `status` - Статус (pending, confirmed, in_progress, completed, cancelled)
- `total_price` - Общая цена
- `notes` - Примечания
- `created_at` - Дата создания
- `updated_at` - Дата обновления

### 4.4. Таблица reviews (Отзывы)
- `id` - Primary Key
- `booking_id` - Foreign Key → bookings.id (уникальный)
- `client_id` - Foreign Key → users.id
- `professional_id` - Foreign Key → users.id
- `rating` - Рейтинг (1-5)
- `comment` - Комментарий
- `created_at` - Дата создания

## 5. API Endpoints (Полный список)

### 5.1. Аутентификация (`/api/auth`)
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `GET /api/auth/me` - Текущий пользователь

### 5.2. Пользователи (`/api/users`)
- `GET /api/users` - Список пользователей
- `GET /api/users/professionals` - Список мастеров
- `GET /api/users/{user_id}` - Профиль пользователя
- `PUT /api/users/me` - Обновить свой профиль

### 5.3. Услуги (`/api/services`)
- `GET /api/services` - Список услуг
- `GET /api/services/categories` - Список категорий
- `GET /api/services/{service_id}` - Детали услуги
- `POST /api/services` - Создать услугу (мастер)
- `PUT /api/services/{service_id}` - Обновить услугу
- `DELETE /api/services/{service_id}` - Деактивировать услугу

### 5.4. Бронирования (`/api/bookings`)
- `GET /api/bookings` - Список бронирований
- `GET /api/bookings/{booking_id}` - Детали бронирования
- `POST /api/bookings` - Создать бронирование
- `PUT /api/bookings/{booking_id}` - Обновить бронирование
- `POST /api/bookings/{booking_id}/cancel` - Отменить бронирование

### 5.5. Отзывы (`/api/reviews`)
- `GET /api/reviews` - Список отзывов
- `GET /api/reviews/{review_id}` - Детали отзыва
- `POST /api/reviews` - Создать отзыв

### 5.6. Кабинет мастера (`/api/professional`)
- `GET /api/professional/stats` - Статистика мастера
- `GET /api/professional/services` - Услуги мастера
- `POST /api/professional/services` - Создать услугу
- `PUT /api/professional/services/{service_id}` - Обновить услугу
- `DELETE /api/professional/services/{service_id}` - Деактивировать услугу
- `GET /api/professional/bookings` - Бронирования мастера
- `PUT /api/professional/bookings/{booking_id}/status` - Изменить статус
- `GET /api/professional/reviews` - Отзывы о мастере

### 5.7. Кабинет клиента (`/api/client`)
- `GET /api/client/stats` - Статистика клиента
- `GET /api/client/bookings` - Бронирования клиента
- `GET /api/client/bookings/{booking_id}` - Детали бронирования
- `GET /api/client/reviews` - Отзывы клиента
- `GET /api/client/favorites/professionals` - Избранные мастера

### 5.8. Админ-панель (`/api/admin`)
- `GET /api/admin/stats` - Статистика системы
- `GET /api/admin/users` - Все пользователи
- `GET /api/admin/users/{user_id}` - Пользователь

- `DELETE /api/admin/users/{user_id}` - Деактивировать пользователя
- `POST /api/admin/users/{user_id}/activate` - Активировать пользователя
- `GET /api/admin/services` - Все услуги
- `DELETE /api/admin/services/{service_id}` - Деактивировать услугу
- `POST /api/admin/services/{service_id}/activate` - Активировать услугу
- `GET /api/admin/bookings` - Все бронирования
- `PUT /api/admin/bookings/{booking_id}` - Обновить бронирование
- `GET /api/admin/reviews` - Все отзывы

4. Порт по умолчанию: 5174
